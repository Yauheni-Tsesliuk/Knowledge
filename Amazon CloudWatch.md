
# Amazon CloudWatch

-------------------------

это сервис мониторинга для ресурсов Amazon Web Services (AWS) и приложений, работающих в облаке, который собирает метрики, журналы, события и предоставляет инструменты для визуализации, оповещения и автоматизации для обеспечения наблюдаемости и производительности ваших систем. Он позволяет отслеживать использование CPU, памяти, диска, сетевой трафик, а также анализировать журналы ошибок и событий для оперативного реагирования на проблемы и масштабирования.

Основные возможности

  

Сбор метрик: Собирает метрики производительности (CPU, память, сеть) и пользовательские метрики в реальном времени.

  

Мониторинг журналов (CloudWatch Logs): Собирает, хранит и анализирует журналы приложений и систем (включая журналы от более чем 30 сервисов AWS и пользовательские журналы).

  

Мониторинг событий (CloudWatch Events/EventBridge): Потоки системных событий, описывающих изменения в ресурсах AWS, для запуска правил и автоматических действий.

  

Оповещения (Alarms): Создание пороговых значений для метрик и событий, которые могут запускать уведомления (Amazon SNS) или автоматические действия (например, масштабирование EC2, запуск Lambda).

  

Автоматизация: Интеграция с другими сервисами для автоматического реагирования на события и изменения состояния.

  

Визуализация и дашборды: Создание настраиваемых панелей для представления данных о состоянии системы.

  

Отслеживание приложений (APM): Функции для мониторинга производительности приложений с поддержкой OpenTelemetry.

  

Для чего используется

  

Для получения полной картины состояния вашей AWS-инфраструктуры и приложений.

Для быстрого выявления и устранения неполадок.

Для управления нагрузкой и автоматического масштабирования ресурсов.

Для обеспечения соответствия требованиям и аудита через анализ журналов.
